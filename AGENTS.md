# リポジトリ運用ルール

## ブランチとワークツリー

- 作業セッションを開始するたびに、専用の Git ブランチを作成すること。
- Git ワークツリーの分割は任意とする。使用する場合は、セッション専用のワークツリーを作成すること。
- セッション内の変更は、そのセッション専用のブランチ上で行うこと。ワークツリーを使用する場合は、そのワークツリー上で行うこと。

## コミット

- `main` ブランチへ直接コミットしてはならない。
- `main` への反映は、セッション用ブランチで変更をコミットした後、レビュー済みのマージまたはプルリクエストを通じて行うこと。

## Issue と進捗ラベル

- セッション開始時に、そのセッションで扱う作業の GitHub Issue を作成すること。
- タスクを他の担当者またはエージェントへ振り分ける際は、振り分けるタスクごとに GitHub Issue を作成すること。
- 各 Issue は、進捗に応じて次のいずれか 1 つのラベルを付与すること。
  - `TODO`: 未着手
  - `InProgress`: 作業中
  - `Done`: 完了
- 作業開始時に `TODO` から `InProgress` へ、完了時に `InProgress` から `Done` へ更新すること。
